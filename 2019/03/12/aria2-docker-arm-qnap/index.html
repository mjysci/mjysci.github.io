ARM平台qnap用Docker部署aria2

笔记
  • arm
  • docker
  • nas

亦适用于其他64位的ARM平台（arm64v8）。

浏览器登录NAS

安装Container Station

App Center -> All Apps -> Container Station -> Install

获取arm版aria2镜像

用ssh登录NAS，然后执行以下命令。

运行aria2容器

将下面命令中的/share/Download/aria2/data和/share/Download/aria2/config换成自己喜欢的位置。
-u换成自己管理员用户的uid，uid可通过id <username>查询。
YOUR_RPC_TOKEN换成自用密码。

登录AriaNg

浏览器访问NAS:<yourip>:6888。
第一次登录会提示认证失败。AriaNg 设置 -> RPC -> Aria2 RPC 密钥处填上前面设置的YOUR_RPC_TOKEN并刷新。

备用

以下为QTS UI设置步骤，但由于不能指定docker run -u的参数，故下载文件权限是admin的，不是很方便。

Create Container -> Advanced Settings下设置容器运行参数

Evironment

添加Name为SECRET的环境变量，Value为你连接aria2要用到的RPC令牌，待会有用。

Network -> Port Forwarding

Container的端口已自动填好，只需填入Host对应的端口，注意不要占用NAS的80和8080端口，如3080，4080。

Shared Folder

此处不要用默认的New volume栏，而是从Volume from host选择容器内/data和/config对应NAS的文件夹。
